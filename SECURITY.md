# Security Policy

codex-sk is early-stage software. Security reports are still taken seriously,
but there is not yet a stable release line or formal vulnerability disclosure
program.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Report vulnerabilities through GitHub private vulnerability reporting when it is
enabled for the repository. If that is not available, contact a maintainer
privately before sharing exploit details in a public issue. Include:

- affected commit or version
- impact summary
- reproduction steps
- any known mitigations
- whether the issue is already public

Do not include secrets, credentials, private keys, or sensitive production data
in reports. If sensitive data is required to explain the issue, describe the
shape of the data instead.

## Response Expectations

Maintainers should acknowledge security reports when available and prioritize
issues based on impact, exploitability, and affected users. Because the project
does not yet have a stable release, fixes currently target the default branch.

## Supported Versions

codex-sk has not published a stable release yet. Security fixes currently target
the default branch.

## Public Discussion

Please allow maintainers time to investigate before publishing details that
could put users at risk.
