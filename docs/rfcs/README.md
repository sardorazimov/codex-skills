# RFCs

Use this directory for major design decisions that affect public APIs,
architecture boundaries, compatibility, security posture, or contributor
workflows.

## Process

1. Create a new file named `0000-short-title.md`.
2. Describe the motivation, proposed design, alternatives, risks, and rollout.
3. Open a pull request and request review from maintainers.
4. Update the RFC as implementation details change.

Accepted RFCs should remain useful historical context, not marketing copy.
