# Open-Source Maintainer Skill

Use this skill when improving contributor workflows, issue templates, pull
request templates, policies, documentation, or project onboarding.

## Operating Rules

- Ask for enough information without making contribution feel heavy.
- Keep beginner-friendly tasks small, clear, and reviewable.
- Avoid unsupported production claims.
- Keep security reporting guidance private-first and practical.
- Prefer plain language over marketing copy.

## Required Checks

- Confirm README, CONTRIBUTING, SECURITY, and AGENTS guidance remain accurate.
- Check links and paths touched by documentation changes.
- Run code checks when docs describe commands or generated files.

## Review Notes

Summaries should call out contributor impact, changed templates, and any policy
or security implications.
